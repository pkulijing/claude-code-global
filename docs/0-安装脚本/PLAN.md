# 安装脚本实现计划

## Context

用户有多台开发设备，需要同步 Claude Code 全局配置（`CLAUDE.md` 和 `skills/`）。已建立 GitHub 仓库，需要一个安装脚本将仓库内容通过符号链接部署到 `~/.claude/`。

## 实现方案

创建 `install.sh`，逻辑如下：

### 1. 基础设置
- 自动检测脚本所在目录作为仓库根目录（`REPO_DIR`）
- 目标目录固定为 `~/.claude/`
- 确保 `~/.claude/` 和 `~/.claude/skills/` 目录存在

### 2. 链接 CLAUDE.md
- 已存在且不是符号链接 → 备份为 `CLAUDE.md.bak`，然后创建软链接
- 已是指向本仓库的正确软链接 → 跳过
- 指向其他位置的软链接 → 替换
- 不存在 → 直接创建

### 3. 链接 skills
- 遍历仓库 `skills/` 下的每个子目录，逐个软链接到 `~/.claude/skills/` 下
- 处理逻辑同 CLAUDE.md
- 不删除不属于本仓库的 skill

### 4. 输出
- 每步操作打印状态信息（已链接 / 已跳过 / 已备份）

## 涉及文件

- 新建：`install.sh`（仓库根目录）

## 验证方式

1. 运行 `bash install.sh`，检查软链接正确
2. 再次运行确认幂等性
3. 手动创建不属于仓库的 skill，运行脚本确认不会被删除
