# 开发总结：安装脚本

## 开发项背景

用户有多台开发设备，希望通过 GitHub 仓库同步 Claude Code 的全局配置（`CLAUDE.md` 和 `skills/`）。需要一个安装脚本将仓库内容部署到各设备的 `~/.claude/` 目录下。

## 实现方案

### 关键设计

采用符号链接方案，而非文件复制：
- `CLAUDE.md` 整文件软链接到 `~/.claude/CLAUDE.md`
- `skills/` 下的每个子目录**逐个**软链接到 `~/.claude/skills/` 下，而非整个 `skills/` 目录做单一软链接，以保留用户在其他来源添加的 skills

### 开发内容概括

创建了 `install.sh` 脚本，包含以下功能：
- 自动检测仓库根目录
- 智能处理已存在的文件/链接（备份、更新、跳过）
- 幂等执行：重复运行不会产生副作用
- 彩色状态输出，清晰展示每步操作结果

### 额外产物

- `docs/0-安装脚本/PROMPT.md`：需求文档
- `docs/0-安装脚本/PLAN.md`：实现计划

## 局限性

- 脚本仅支持单向同步（仓库 → 本地），如果用户在 `~/.claude/` 下直接修改了被链接的文件，修改会反映到仓库中（符号链接的特性），但需要手动 commit/push
- 没有提供卸载/清理功能（移除软链接、恢复备份）
