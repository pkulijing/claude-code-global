# 实现计划：/devtree skill

## 分类体系设计

6 类，按「改动了什么」+「目的是什么」两个维度划分：

| 类型 | 图标 | 判断标准 |
|------|------|---------|
| 初建 | 🌱 | 某功能域首次从零建立 |
| 功能 | ✨ | 扩展用户可感知的能力 |
| 修复 | 🐛 | 纠正缺陷或回归 |
| 重构 | 🏗️ | 内部结构改善，用户行为不变 |
| 工程 | 📦 | 打包/CI/分发/工具链 |
| 探索 | 🔬 | 调研，可能被搁置或回退 |

重构 vs 工程的关键区别：重构改源代码结构，工程改构建/分发/CI 链路。

## 父节点策略

- 每个节点只有一个父节点（严格树结构，不允许多父）
- 选「最直接的上游」，不按时间顺序
- 并行分支追溯到共同的真实前序节点

## 文件结构

- 输出文件：`docs/DEVTREE.md`
- 格式：Mermaid `graph TD` + 节点索引表
- 节点 ID：`N{编号}`，按 classDef 配色区分类型

## 实现步骤

1. 新建 `skills/devtree/SKILL.md`，包含：
   - 分类体系说明
   - 父节点选择原则
   - 重建模式（首次）vs 增量模式（已有 DEVTREE.md）的完整执行流程
   - DEVTREE.md 的格式规范（含 Mermaid 模板和节点索引表）

2. 修改 `skills/finish/SKILL.md`，在写 SUMMARY.md 后、`/commit` 前插入步骤：调用 `/devtree` 更新开发树

3. 运行 `bash install.sh` 安装新 skill（由用户手动执行）
