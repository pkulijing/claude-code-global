# 需求：创建 /devtree skill

## 背景

随着项目开发历史变长，`docs/` 下的编号文件夹越来越多，简单的线性编号无法清晰表达开发项之间的时间和逻辑关系。希望有一个图形化的宏观文档，把所有开发项的分类与逻辑父子关系可视化为树状图。

## 需求描述

新增一个全局 skill `/devtree`，在完成一个开发项（写完 SUMMARY.md）即将提交代码之前，自动维护项目根目录下的 `docs/DEVTREE.md`。

**功能要求**：
1. 为每个开发项分配类型（初建/功能/修复/重构/工程/探索）
2. 语义分析确定父节点，不按时间顺序强行构造父子关系——并行的功能分支追溯到共同的真正前序节点
3. 每个节点只有一个父节点，保持严格树结构
4. 使用 Mermaid 图表可视化，配色区分分类
5. 对无此图的历史项目，从头扫描所有 SUMMARY.md 重建完整树
6. 集成到 `/finish` 流程中，自动触发
