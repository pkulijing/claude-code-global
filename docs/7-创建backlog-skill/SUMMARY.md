# 第 7 轮开发总结：创建 /backlog skill

## 开发项背景

第 6 轮给本仓库引入了 `docs/BACKLOG.md` 用于登记待启动开发项，同时 `/finish` skill 承担了「开发项来自 BACKLOG 时顺手删除对应条目」的职责。但**添加**侧是个空白：

- 没有工具支持"向 BACKLOG 追加条目"——每次都要手动编辑
- 没有统一的条目模板约束，未来其他项目各自为战
- 第 6 轮临时写的 `docs/BACKLOG.md` 开头过于简略，缺乏「本文件是权威来源 / 工作流 / 条目没有固定优先级」这些通用说明

手边有一份已经验证过的样本：`whisper-input/BACKLOG.md` 的开头，那套"动机 / 目标状态 / 候选方向 / 风险 / scope"模板和"条目没有固定优先级"的方法论是跨项目通用的，值得抽象出来复用。

## 实现方案

### 关键设计

1. **skill 不做纯搬运，做交互式扩写 + 归类**：初版 PLAN 设想只是"参数原样落盘"，review 时调整为：
   - **扩写阶段**：拿到用户输入后按五字段模板给结构化草稿，缺的字段写 `待补充` 而不是脑补，避免放大 LLM 编造倾向
   - **归类阶段**：读现有 `##` 分类推荐归属（或建议新建分类）
   - **确认阶段**：草稿和归类建议一并展示，人类确认后才落盘
   - 这样 skill 的价值不是 `echo >> file`，而是强制一次"写清楚再加"的仪式
2. **占位豁免**：骨架原文鼓励"哪怕只写一行占位，之后再补完整"。如果用户明确说"只写一行占位"，skill 尊重意图、跳过扩写——模板是帮助不是枷锁
3. **删除职责不重复**：明确声明 skill 只管添加，删除走 `/finish`；避免两个 skill 互相漂移出对方的边界
4. **骨架通用化**：相对 whisper-input 原版，标题从 `# Whisper Input — Backlog` 改为 `# Backlog`，删掉手工维护的 `## 目录`（自动同步目录会让 skill 职责膨胀），其余骨架文字原样继承
5. **不自动 commit**：`/backlog` 结束后让用户自行决定提交时机，符合"轻量追加"定位

### 开发内容概括

- 新增 `skills/backlog/SKILL.md`：五阶段流程（扩写 → 归类 → 确认 → 落盘 → 反馈），含骨架模板和全部交互原则
- 重写 `docs/BACKLOG.md`：替换第 6 轮临时开头为新骨架，把原有「Permission 配置」条目按五字段结构化，归入新分类 `## 配置治理`，作为 skill 产出的「正确长相」样本
- 无需改动 `install.sh`：其 `for skill_dir in skills/*/` 循环天然覆盖新增 skill

### 额外产物

- `docs/7-创建backlog-skill/PROMPT.md`、`PLAN.md`——走完完整的四步流程，PLAN 过程中经过一次 review 反馈（"分类要推荐让人确认 / 内容要扩写不要原样搬"）修正后才执行

## 局限性

- **骨架 vs 已有自定义开头**：skill 只在文件不存在时写骨架。如果某项目已手写了别的开头，skill 不会覆盖——不同项目的 BACKLOG 开头可能长期长得不一样，这是可接受的折中
- **分类在 skill 里是 flat 的**：归类只看一级 `## ` section，不处理 `### ` 下再嵌一层（whisper-input 原文的结构实际就是这样，目前没遇到真需要嵌套的场景）
- **并发追加不保护**：两次快速连续调用理论上有竞态；罕见且损坏面小，失败重试即可
- **扩写只跑一次**：用户在确认阶段提调整时，skill 会按指令改字段，但不会再触发新一轮"基于修改后的上下文重新扩写"——改动小时没问题，改动大时需要用户自己 /backlog 重来
- **还没跑 `install.sh` 部署**：本仓库的 `~/.claude/skills/backlog/` 软链接待重新安装才能生效（项目 CLAUDE.md 已有相关说明）

## 后续 TODO

- `/backlog` 真正被用起来后，观察"扩写生成的草稿被人类大幅修改"的比例——如果修改率太高说明模板字段不对、或者 AI 扩写质量不够，届时再迭代
- 是否需要 `/backlog --raw <content>` 的一键裸追加模式，跳过扩写和归类？当前通过"用户说'只写一行占位'"覆盖了这个场景，但如果用多了发现有相当比例的条目确实就是一行占位，值得做个显式 flag
- 其他项目（whisper-input 等）已有的 BACKLOG.md 需要人工迁移到 `docs/BACKLOG.md` 位置，这是用户自己要做的事，不在本轮 scope 内
